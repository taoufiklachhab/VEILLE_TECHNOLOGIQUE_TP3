RPi Home Storage

Présentation du projet

Transformer un Raspberry Pi en serveur de stockage domestique pour centraliser fichiers et images, comme un Google Drive personnel mais auto-hébergé, économique et privé.

Serveur de fichiers

Type Google Drive perso

Stockage d'images

Avec galerie intégrée

Accessible depuis ton réseau

Local uniquement

Architecture visée : Raspberry Pi OS + Docker + Nextcloud (fichiers) + PhotoPrism (photos) + disque dur externe + LEDs GPIO.

Projet réalisé dans le cadre d’expérimentations matérielles pour valider la faisabilité d’un serveur de stockage domestique low-cost.

Expérimentations matérielles

Exp. 1 : Installation et boot du Raspberry Pi 4

Technologies : Raspberry Pi 4, microSD 64GB, alimentation USB-C

Objectif : Vérifier que le RPi4 démarre correctement et supporte un usage serveur 24/7 sans surchauffe.

Contexte : Flashage de Raspberry Pi OS Lite sur microSD via Raspberry Pi Imager. Connexion HDMI pour premier boot, puis passage en SSH. Température mesurée via vcgencmd measure_temp toutes les heures pendant 6h.

Résultat : Boot réussi en 25 secondes. Température au repos : 42°C. En charge (installation de Docker) : 58°C. Stabilité parfaite.

Avis : Puissance suffisante pour un serveur de stockage domestique. Limite : sans ventilateur, température monte à 65°C en été. Potentiel excellent pour le coût.

Avis final : Validation de l’hypothèse — Le RPi4 peut servir de serveur de stockage domestique sans surchauffe problématique.

Exp. 2 : Montage et accès au SSD externe

Technologies : SSD Samsung T7, port USB 3.0 du RPi4

Objectif : Valider la détection, le montage automatique et la vitesse de transfert du SSD.

Contexte : Connexion du SSD au port USB 3.0. Formatage en ext4. Montage dans /mnt/storage. Test de vitesse avec dd et hdparm. Copie d’un fichier de 5 Go depuis un ordinateur via Samba.

Résultat : Lecture séquentielle : 380 MB/s, écriture : 340 MB/s. Montage automatique réussi via fstab. Pas de déconnexion intempestive.

Avis : Le SSD externe est un énorme plus pour la réactivité et la fiabilité. Limite : le port USB partage la bande passante avec Ethernet, mais reste acceptable.

Avis final : Validation de l’hypothèse — Le SSD est parfaitement adapté et recommandé pour tout serveur domestique sur RPi.

Exp. 3 : Test de Nextcloud sur RPi4 avec Docker

Technologies : RPi4, Docker, Nextcloud, SSD externe

Objectif : Vérifier que Nextcloud tourne correctement et permet l’upload/partage de fichiers.

Contexte : Installation de Docker + Docker Compose. Fichier compose avec Nextcloud + MariaDB + Redis. Montage du SSD comme volume de données. Accès via navigateur depuis un PC du même réseau.

Résultat : Nextcloud fonctionne fluide pour 1 utilisateur. Upload d’un fichier de 2 Go en 35 secondes. Consommation CPU max à 70% pendant l’upload.

Avis : Nextcloud est un peu lourd pour le RPi4, mais tout à fait utilisable. Limite : plusieurs utilisateurs simultanés pourraient ralentir le système.

Avis final : Validation de l’hypothèse — Nextcloud est viable pour un usage personnel ou petite famille.

Exp. 4 : LEDs GPIO - Vert (ajout) / Rouge (suppression)

Technologies : RPi4, LED Verte (GPIO17), LED Rouge (GPIO27), résistances 330Ω

Objectif : Créer un feedback visuel physique : LED verte clignote lors d’un ajout de fichier, LED rouge clignote lors d’une suppression.

Contexte : Branchement des LEDs sur les broches GPIO avec résistances. Script Python utilisant la librairie watchdog pour surveiller le dossier /mnt/storage. À chaque création de fichier → LED verte clignote 3 fois. À chaque suppression → LED rouge clignote 3 fois.

Script Python (gpio_notify.py) :
import RPi.GPIO as GPIO
import time
from watchdog.observers import Observer
from watchdog.events import FileSystemEventHandler

LED_VERT = 17
LED_ROUGE = 27

GPIO.setmode(GPIO.BCM)
GPIO.setup(LED_VERT, GPIO.OUT)
GPIO.setup(LED_ROUGE, GPIO.OUT)

def blink(pin, times=3, speed=0.2):
for _ in range(times):
GPIO.output(pin, True)
time.sleep(speed)
GPIO.output(pin, False)
time.sleep(speed)

class Handler(FileSystemEventHandler):
def on_created(self, event):
blink(LED_VERT) # Vert clignote à l'ajout
def on_deleted(self, event):
blink(LED_ROUGE) # Rouge clignote à la suppression

observer = Observer()
observer.schedule(Handler(), "/mnt/storage", recursive=True)
observer.start()
try:
while True:
time.sleep(1)
except KeyboardInterrupt:
observer.stop()
observer.join()
GPIO.cleanup()

Résultat : Les LEDs réagissent instantanément aux événements fichiers (création/suppression). Testé avec copie/suppression manuelle et via Nextcloud. Consommation CPU négligeable (~1%).

Avis : Solution très efficace pour un feedback utilisateur physique. Vert clignotant → ajout réussi. Rouge clignotant → suppression effectuée. Limite : nécessite un script en arrière-plan (systemd).

Avis final : Validation de l’hypothèse — Les LEDs GPIO offrent un retour visuel clair et satisfaisant pour les opérations d’ajout/suppression.

Exp. 5 : PhotoPrism (galerie photo) sur RPi4

Technologies : RPi4, Docker, PhotoPrism, SSD

Objectif : Tester PhotoPrism pour la gestion de photos.

Contexte : Installation via Docker Compose officiel PhotoPrism. Upload de 100 photos. Activation de l’indexation automatique.

Résultat : PhotoPrism tourne correctement avec des ressources modérées. Indexation de 100 photos en 3 minutes.

Avis : PhotoPrism est bien adapté au RPi4. Interface propre, recherche par date/lieu.

Avis final : Validation de l’hypothèse — PhotoPrism est une excellente solution de galerie photo pour RPi4.
